# <a name="set-up-fraud-alerts"></a><span data-ttu-id="1b0b5-104">Konfigurowanie alertów o oszustwie</span><span class="sxs-lookup"><span data-stu-id="1b0b5-104">Set up fraud alerts</span></span>

[!include [banner](includes/banner.md)]

<span data-ttu-id="1b0b5-105">W tym temacie opisano sposób ustawiania kryteriów i zasad w celu wstrzymania potencjalnie fałszywych zamówień sprzedaży w celu dalszego sprawdzenia.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-105">This topic explains how to set up criteria and rules to place potentially fraudulent sales orders on hold for further review.</span></span> <span data-ttu-id="1b0b5-106">Funkcja wykrywania oszustw służy do określania prawidłowości informacji w zamówieniu sprzedaży.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-106">Fraud review functionality is used to determine the validity of information in a sales order.</span></span> <span data-ttu-id="1b0b5-107">Jeżeli informacje w zamówieniu sprzedaży budzą wątpliwości na podstawie obowiązujących w organizacji kryteriów i zasad dotyczących oszustw, zamówienie może zostać wstrzymane w celu dalszego sprawdzenia przez administratora.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-107">If the information in the sales order appears to be questionable based on an organization’s fraud criteria and rules, the order may be put on hold for further review by an administrator.</span></span>

> [!NOTE]
> <span data-ttu-id="1b0b5-108">Tej funkcji można używać tylko z przetwarzaniem zamówień sprzedaży kanału Biuro obsługi rozwiązania Retail.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-108">This feature can only be used with Retail Call Center channel sales order processing.</span></span> 

<span data-ttu-id="1b0b5-109">Po zdefiniowaniu kanału Biuro obsługi należy ustawić opcję **Włącz kończenie zamówienia** na **Tak**.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-109">When the Call Center channel is defined, **Enable Order Completion** must be set to **Yes**.</span></span> <span data-ttu-id="1b0b5-110">Po włączeniu kończenia zamówienia użytkownicy mogą wyświetlić podsumowanie zamówienia i kliknąć opcję **Prześlij**, aby sfinalizować zamówienie.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-110">When order completion is enabled, users can view the order recap and click **Submit** to finalize the order.</span></span> <span data-ttu-id="1b0b5-111">Użytkownicy będą także mogli korzystać z opcji ręcznego wstrzymania fałszywego zamówienia sprzedaży w celu sprawdzenia.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-111">Users will also be provided options to manually place the sales order on fraud review hold.</span></span> <span data-ttu-id="1b0b5-112">Zamówienia sprzedaży wprowadzone przez użytkownika Centrum obsługi są przetwarzane z użyciem zasad i kryteriów wykrywania oszustw podczas procesu przesyłania.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-112">Sales orders keyed in by a Call Center user are processed through fraud check rules and criteria during the submit process.</span></span>

<span data-ttu-id="1b0b5-113">Istnieją dwa typy kryteriów wykrywania oszustw, które system wykorzysta do sprawdzenia, czy należy wstrzymać zamówienie w celu sprawdzenia, czy nie stanowi oszustwa:</span><span class="sxs-lookup"><span data-stu-id="1b0b5-113">There are two types of fraud criteria that the system will reference to check if the order should be held for fraud review:</span></span>

-   <span data-ttu-id="1b0b5-114">**Reguły statyczne** wykorzystują określoną wartość, na przykład numer telefonu umieszczony na czarnej liście lub adres e-mail oznaczony jako znany z wcześniejszego wykorzystania do przeprowadzenia transakcji związanych z oszustwem.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-114">**Static rules** use a specific value, such as a phone number that has been blacklisted or an email address that has been flagged as being known as used for past fraudulent transactions.</span></span> <span data-ttu-id="1b0b5-115">Na stronie **Fałszywe dane statyczne** można dodać fałszywe informacje ręcznie lub za pomocą importu danych i dołączyć do nich punktację.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-115">On the **Static Fraud Data** page, fraud information can be added manually or through data import, with scores attached to the fraudulent information.</span></span> <span data-ttu-id="1b0b5-116">Jeżeli wykrywanie oszustwa jest włączone, każde wprowadzone zamówienie sprzedaży zostanie porównane z danymi statycznymi.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-116">If fraud checking is turned on, every sales order entered will be compared to the static data.</span></span> <span data-ttu-id="1b0b5-117">Jeżeli dane zostaną znalezione w adresie fakturowania lub dostawy lub w adresie dostawy w wierszu sprzedaży, punktacja wszystkich unikatowych dopasowań zostanie zsumowana.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-117">If the data is found in either the customer’s billing or delivery address, or if the data is found in the delivery address on the sales line, the scores of all unique matches will be summed.</span></span>  
-   <span data-ttu-id="1b0b5-118">**Reguły dynamiczne** składają się ze zmiennych i warunków zdefiniowanych dla tych zmiennych.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-118">**Dynamic rules** are composed from variables and conditions defined for those variables.</span></span> <span data-ttu-id="1b0b5-119">Za pomocą reguł dynamicznych można sprawdzić inne kryteria niezdefiniowane w regułach statycznych.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-119">With dynamic rules, other criteria not defined in the static rules can be checked.</span></span> <span data-ttu-id="1b0b5-120">Bardziej złożone instrukcje „ORAZ/LUB” umożliwiają uwzględnienie wielu warunków do określenia, czy istnieje pozytywne dopasowanie względem kryteriów reguły i przesłanego zamówienia sprzedaży.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-120">More complex “AND/OR” statements can be used to consider multiple conditions to determine if there is a positive match against the rule criteria and the sales order being submitted.</span></span> <span data-ttu-id="1b0b5-121">Jeżeli na przykład użytkownik chce wstrzymać w celu sprawdzenia pod kątem oszustwa wszystkie zamówienia klientów związanych z określoną wartością grupy klientów i którzy zamówili określony produkt, warunki sprawdzenia poprawności klienta i produktów zostaną zdefiniowane na stronie **Reguły** z warunkiem „ORAZ”.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-121">For example, if a user wants to hold for fraud review all orders for customers that are tied to a specific customer group value and that ordered a specific product, the conditions to validate customer and validate products would be defined on the **Rules** page with an “AND” condition.</span></span> <span data-ttu-id="1b0b5-122">Zamówienie zostanie wstrzymane tylko jako fałszywe tylko jeżeli oba warunki są prawdziwe, a wartość punktacji przypisana do reguły spowoduje ustawienie całkowitej punktacji zamówienia dotyczącej oszustwa powyżej wartości minimalnej zdefiniowanej w parametrach biura obsługi.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-122">The order would fall into fraud hold only if both conditions were true and if the score value assigned to the rule put the order’s total fraud score over the minimum fraud score as defined in Call Center parameters.</span></span>

<span data-ttu-id="1b0b5-123">Użytkownik biura obsługi może także ręcznie dodać zamówienie do kolejki wstrzymania z powodu oszustwa.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-123">A call center user may also manually place an order into the fraud hold queue.</span></span> <span data-ttu-id="1b0b5-124">Jeżeli użytkownik wprowadzający zamówienie uważa, że klient składający zamówienie jest podejrzany i chce, aby inna osoba dodatkowo sprawdziła zamówienie przed przetworzeniem, użytkownik wprowadzający zamówienie może wybrać opcję **Ręczne wstrzymanie z powodu oszustwa** z listy rozwijanej **Wstrzymania** na stronie **Podsumowanie zamówienia sprzedaży** (wyświetlana po wywołaniu funkcji **Zakończ zamówienie**).</span><span class="sxs-lookup"><span data-stu-id="1b0b5-124">If the user entering the order believes the customer placing the order may be suspicious and wants someone else to further review the validity of the order before it is processed, the user entering the order can choose the **Manual Fraud Hold** option from the **Holds** dropdown menu on the **Sales Order Summary** page (this appears after the **Complete** order function is called).</span></span> <span data-ttu-id="1b0b5-125">Zostanie wyświetlony monit o wprowadzenie przez użytkownika notatki z dodatkowym wyjaśnienie dlaczego ich zdaniem zamówienie może być próbą oszustwa, aby osoba sprawdzająca miała dodatkowe informacje.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-125">The user will be prompted to enter a note to further explain why they feel the order may be fraudulent so that the reviewer has more context.</span></span>

<span data-ttu-id="1b0b5-126">Wszystkie zamówienia wstrzymane za pomocą ręcznego wstrzymania z powodu oszustwa lub systematycznego obliczania punktacji dotyczącej oszustwa będą widoczne na stronie **Wstrzymania zamówień**, gdzie można sprawdzić zamówienie, a następnie anulować je lub zwolnić do przetwarzania.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-126">All orders held through manual fraud hold or by systematic calculation of fraud scores will appear on the **Order Holds** page, where the order can be reviewed and then either canceled or released to processing.</span></span>

> [!NOTE]
> <span data-ttu-id="1b0b5-127">Użycie wielu reguł lub zbyt złożonych reguł spowoduje ograniczenie wydajności systemu po przesłaniu zamówień sprzedaży.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-127">Use of multiple rules or overly complex rules will result in poor system performance when sales orders are submitted.</span></span> <span data-ttu-id="1b0b5-128">Funkcja fraud alert feature hasn't been optimized to handle a large volume of static fraud data entries and many active rules.</span></span> <span data-ttu-id="1b0b5-129">Należy pamiętać, że każda reguła jest oceniana podczas przesyłania wpisu zamówienia sprzedaży biura obsługi.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-129">It’s important to remember that every rule is evaluated during the submit function of call center sales order entry.</span></span> <span data-ttu-id="1b0b5-130">Te reguły są oceniane względem nagłówka zamówienia sprzedaży i wszystkich wierszy zamówień.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-130">The rules are evaluated against the sales order header and all order lines.</span></span> <span data-ttu-id="1b0b5-131">Im więcej zasad i złożoność instrukcji reguły tym dłuższy czas przetwarzania.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-131">The more rules there are and the more complex the rule statements are, the longer it will take to process.</span></span> <span data-ttu-id="1b0b5-132">Jeżeli masz w zamówieniu dużą liczbę pozycji w wierszu, dużą liczbę aktywnych reguł i statycznych wpisów danych, systematyczny proces przeglądania i sprawdzania poprawności wszystkich danych i obliczania punktacji dotyczącej oszustwa może mieć duży wpływ na wydajność.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-132">If you have a large number of line items on your order, and a large number of active rules and static data entries, this systematic process of reviewing and validating all of the data and calculating a fraud score can have a severe performance impact.</span></span>  <span data-ttu-id="1b0b5-133">Organizacje korzystające z tej funkcji powinny zawsze przetestować i sprawdzić, czy czas przetwarzania przesyłania zamówień jest dopuszczalny przed zastosowaniem zmian w regułach lub statycznych kryteriach dotyczących oszustwa w środowisku produkcyjnym.</span><span class="sxs-lookup"><span data-stu-id="1b0b5-133">Organizations using this feature should always test and confirm that the order submission processing time is acceptable before applying any changes to rules or static fraud criteria into the production environment.</span></span>
