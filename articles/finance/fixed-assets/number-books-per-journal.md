# <a name="number-of-books-per-journal"></a><span data-ttu-id="8731f-104">Liczba ksiąg na arkusz</span><span class="sxs-lookup"><span data-stu-id="8731f-104">Number of books per journal</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="8731f-105">W tym temacie opisano relacje między arkuszami i księgami składników majątku występujące podczas tworzenia propozycji nabycia lub amortyzacji środka trwałego za pomocą zadania wsadowego.</span><span class="sxs-lookup"><span data-stu-id="8731f-105">This topic describes the relationship between journals and asset books when you create a fixed asset acquisition or depreciation proposal through a batch job.</span></span> <span data-ttu-id="8731f-106">Można zdefiniować maksymalną liczbę ksiąg, które są uwzględniane w każdym nabyciu i amortyzacji, używając pól znajdujących się w sekcji **Liczba ksiąg na arkusz** na karcie **Ogólne** na stronie **Parametry środków trwałych** (**Środki trwałe \> Ustawienia \> Parametry środków trwałych**).</span><span class="sxs-lookup"><span data-stu-id="8731f-106">You can define the maximum number of books that are included for each acquisition and for depreciation by using the fields in the **Number of books per journal** section on the **General** tab of the **Fixed assets parameters** page (**Fixed assets \> Setup \> Fixed assets parameters**).</span></span> <span data-ttu-id="8731f-107">Te pola umożliwiają rozdzielenie wszystkich używanych ksiąg składników majątku z podziałem na arkusze nabycia i arkusze amortyzacji.</span><span class="sxs-lookup"><span data-stu-id="8731f-107">These fields let you distribute the number of asset books per acquisition journal and depreciation journal.</span></span>

<span data-ttu-id="8731f-108">W przypadku propozycji nabycia wartość domyślna wynosi co najmniej 10 000 ksiąg.</span><span class="sxs-lookup"><span data-stu-id="8731f-108">For an acquisition proposal, the default value is at least 10,000 books.</span></span> <span data-ttu-id="8731f-109">W przypadku propozycji amortyzacji wartość domyślna wynosi co najmniej 2000 ksiąg.</span><span class="sxs-lookup"><span data-stu-id="8731f-109">For a depreciation proposal, the default value is at least 2,000 books.</span></span>

<span data-ttu-id="8731f-110">Jeśli na przykład istnieje 4000 środków trwałych, a z każdym środkiem trwałym są skojarzone dwie księgi, jedna księga zostanie zaksięgowana w bieżącej warstwie, a druga księga zostanie zaksięgowana w warstwie podatkowej.</span><span class="sxs-lookup"><span data-stu-id="8731f-110">For example, if there are 4,000 fixed assets, and two books are associated with each asset, one book will be posted to the current layer, and the other book will be posted to the tax layer.</span></span> <span data-ttu-id="8731f-111">Jeśli w wyniku przetwarzania wsadowego zostanie nabytych 4000 środków trwałych, to zadanie wsadowe, które utworzyło jeden arkusz nabycia środków trwałych, będzie zawierało 4000 ksiąg składników majątku.</span><span class="sxs-lookup"><span data-stu-id="8731f-111">If you acquire 4,000 fixed assets through batch processing, the batch job that creates one fixed asset acquisition journal will contain 4,000 asset books.</span></span>

> [!NOTE]
> <span data-ttu-id="8731f-112">Utworzony arkusz będzie używany aż do zakończenia zadania wsadowego.</span><span class="sxs-lookup"><span data-stu-id="8731f-112">The journal that is created will continue to be used until the batch job is completed.</span></span>
>
> <span data-ttu-id="8731f-113">Powstałe pochodne księgi nie są wliczane do maksymalnej dozwolonej liczby ksiąg na arkusz.</span><span class="sxs-lookup"><span data-stu-id="8731f-113">The derived books aren't included in the maximum number of books per journal.</span></span>

<span data-ttu-id="8731f-114">Przetwarzania wsadowego można użyć do wykonania amortyzacji dla tego samego zbioru nabytych środków trwałych.</span><span class="sxs-lookup"><span data-stu-id="8731f-114">You can use  batch processing to run depreciation for the same set of acquired assets.</span></span> <span data-ttu-id="8731f-115">Na przykład zadanie wsadowe tworzy dwa arkusze amortyzacji, z których każdy zawiera 2000 ksiąg składników majątku.</span><span class="sxs-lookup"><span data-stu-id="8731f-115">For example, a batch job creates two depreciation journals, each of which consists of 2,000 asset books.</span></span> <span data-ttu-id="8731f-116">Pierwszy arkusz będzie zawierał księgi skojarzone ze środkami trwałymi o numerach od 1 do 2000.</span><span class="sxs-lookup"><span data-stu-id="8731f-116">The first journal will contain books that are associated with the fixed assets that are numbered 1 through 2,000.</span></span> <span data-ttu-id="8731f-117">Drugi arkusz będzie następnie zawierał księgi skojarzone ze środkami trwałymi o numerach od 2001 do 4000.</span><span class="sxs-lookup"><span data-stu-id="8731f-117">The second journal will then contain books that are associated with the fixed assets that are numbered 2,001 through 4,000.</span></span>

<span data-ttu-id="8731f-118">Zadanie przetwarzania wsadowego wyklucza zamknięte księgi.</span><span class="sxs-lookup"><span data-stu-id="8731f-118">The batch processing job excludes closed books.</span></span> <span data-ttu-id="8731f-119">Na przykład w zadaniu wsadowym amortyzacji 10 z pierwszych 2000 ksiąg jest zamkniętych.</span><span class="sxs-lookup"><span data-stu-id="8731f-119">For example, in a batch job for depreciation, 10 of the first 2,000 books are closed.</span></span> <span data-ttu-id="8731f-120">W tym przypadku pierwszy arkusz będzie zawierał księgi skojarzone ze środkami trwałymi o numerach od 1 do 2011.</span><span class="sxs-lookup"><span data-stu-id="8731f-120">In this case, the first journal will contain books that are associated with the fixed assets that are numbered 1 through 2,011.</span></span> <span data-ttu-id="8731f-121">Drugi arkusz będzie następnie zawierał księgi skojarzone ze środkami trwałymi o numerach od 2,012 do 4,000.</span><span class="sxs-lookup"><span data-stu-id="8731f-121">The second journal will then contain books that are associated with the fixed assets that are numbered 2,012 through 4,000.</span></span>

> [!NOTE]
> <span data-ttu-id="8731f-122">Jeśli masz identyfikatory środków trwałych z różnymi separatorami (takimi jak - lub /) i tworzysz transakcje środków trwałych w zadaniach wsadowych, musisz uruchomić osobne zadanie wsadowe dla każdego typu separatora.</span><span class="sxs-lookup"><span data-stu-id="8731f-122">If you have fixed asset IDs with different separators (such as – or /), and you create fixed asset transactions in batch jobs, you must run a separate batch job for each type of separator.</span></span> <span data-ttu-id="8731f-123">System nie może przetwarzać różnych separatorów w ramach tego samego zadania wsadowego.</span><span class="sxs-lookup"><span data-stu-id="8731f-123">The system can't process different separators within the same batch job.</span></span>

<span data-ttu-id="8731f-124">Limit liczby ksiąg jest stosowany, jeśli w tym samym arkuszu nie istnieją zduplikowane identyfikatory składników majątku.</span><span class="sxs-lookup"><span data-stu-id="8731f-124">The limit on the number of books is applied if duplicate asset IDs don't exist in the same journal.</span></span> <span data-ttu-id="8731f-125">Jeśli jednak identyfikator składnika majątku jest taki sam, jak identyfikator księgi, liczba ksiąg na arkusz może zostać przekroczona w celu utrzymania identyfikatora składnika majątku w tym samym arkuszu.</span><span class="sxs-lookup"><span data-stu-id="8731f-125">However, if the asset ID is the same as the book ID, the number of books per journal can be exceeded to keep the asset ID in the same journal.</span></span>

<span data-ttu-id="8731f-126">Na przykład istnieje 5001 identyfikatory środków trwałych, z każdym identyfikatorem środka trwałego są powiązane trzy księgi, a każda księga składników majątku jest księgowana w tej samej warstwie księgowania.</span><span class="sxs-lookup"><span data-stu-id="8731f-126">For example, there are 5,001 fixed asset IDs, three books are associated with each fixed asset ID, and each asset book is posted to the same posting layer.</span></span> <span data-ttu-id="8731f-127">Wykonujesz amortyzacja przez trzy kolejne miesiące bez podsumowania.</span><span utworzony za pomocą zadania wsadowego, a system utworzy siedem arkuszy, które mają 667 identyfikatorów składników majątku, i trzy księgi dla każdego identyfikatora środka trwałego.</span><span class="sxs-lookup"><span data-stu-id="8731f-128">The depreciation journal will be created through a batch job, and the system will create seven journals that have 667 fixed asset IDs and three books for each fixed asset ID.</span></span> <span data-ttu-id="8731f-129">Ogółem powstanie 2001 ksiąg.</span><span class="sxs-lookup"><span data-stu-id="8731f-129">The result will be 2,001 books.</span></span> <span data-ttu-id="8731f-130">W związku z tym w ciągu trzech miesięcy na potrzeby zachowania tych samych identyfikatorów składników majątku w tym samym arkuszu powstaną 6003 wiersze arkuszy.</span><span class="sxs-lookup"><span data-stu-id="8731f-130">Therefore, in three months, there will be 6,003 journal lines to maintain the same asset IDs in the same journal.</span></span> <span data-ttu-id="8731f-131">System utworzy także jeden arkusz, który ma 332 identyfikatory środków trwałych, i trzy księgi dla każdego identyfikatora środka trwałego.</span><span class="sxs-lookup"><span data-stu-id="8731f-131">The system will also create one journal that has 332 fixed asset IDs and three books for each fixed asset ID.</span></span> <span data-ttu-id="8731f-132">W ciągu trzech miesięcy powstanie 2988 wierszy.</span><span class="sxs-lookup"><span data-stu-id="8731f-132">In three months, there will be 2,988 lines.</span></span>

> [!NOTE] 
> <span data-ttu-id="8731f-133">Jeśli parametr **Podsumuj amortyzację** jest włączony podczas tworzenia propozycji amortyzacji, wartość w polu **Liczba ksiąg na arkusz** — propozycja amortyzacji nie ma żadnego wpływu.</span><span class="sxs-lookup"><span data-stu-id="8731f-133">If the **Summarize depreciation** parameter is turned on when you're creating a depreciation proposal, then the value in the **Number of books per journal - Depreciation proposal** field has no effect.</span></span> <span data-ttu-id="8731f-134">W tym przypadku liczba ksiąg na arkusz wynosi 6000, co jest limitem zdefiniowanym wewnętrznie.</span><span class="sxs-lookup"><span data-stu-id="8731f-134">In this case number of books per journal is 6000, which is the internal defined limit.</span></span>
